# КОНТЕКСТ
Тебе нужно провести аудит набора требований к системе. У меня есть строгий внутренний стандарт качества, который включает 14 ключевых свойств и 44 правила написания требований.

# ЗАДАЧА
Проанализируй каждое требование из списка `[ТРЕБОВАНИЯ ДЛЯ АНАЛИЗА]` на соответствие свойствам и правилам

Для каждого отдельного требования выполни следующие шаги:
1.  **Проверка:** Последовательно проверь требование на соответствие каждому из 44 правил и 14 свойств.
2.  **Идентификация проблем:** Если требование нарушает какое-либо правило или не соответствует свойству, четко укажи номер и название нарушенного правила/свойства.
3.  **Объяснение:** Кратко, но емко объясни, в чем именно заключается несоответствие.
4.  **Рекомендация по улучшению:** Предложи конкретную, переформулированную версию требования, которая устраняет все найденные недостатки.
5.  **Оценка:** Предоставь итоговую оценку по каждому требованию в виде структурированного отчета.
